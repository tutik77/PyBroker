# PyBroker — Брокер сообщений на Python

## Описание проекта

PyBroker — брокер сообщений, реализующий паттерны **Pub/Sub** и **Message Queue** с протоколом **STOMP 1.2**.

Брокер обеспечивает асинхронную коммуникацию между микросервисами: издатели публикуют сообщения, подписчики получают их в реальном времени. Поддерживаются два режима доставки — через топики (всем подписчикам) и через очереди (одному из потребителей).

## Цели

- Реализовать собственный брокер сообщений с нуля
- Обеспечить асинхронную коммуникацию между тремя тестовыми микросервисами (1 publisher, 2 subscriber)
- Использовать стандартный протокол STOMP 1.2 для взаимодействия клиентов с брокером
- Предоставить клиентский SDK на Python для удобной работы с брокером

## Требования

### Базовая функциональность
- Реализация паттерна Publisher/Subscriber
- Поддержка очередей сообщений (FIFO, LIFO)
- Публикация сообщений в топики и очереди
- Подписка на топики и очереди
- Обработка множественных подписчиков на один топик/очередь

### Персистентность
- Сохранение сообщений очередей в SQLite
- Восстановление состояния после перезапуска брокера
- Гарантия доставки at-least-once (двухфазная доставка с ACK и таймаутами)

### Клиентская библиотека
- Асинхронный SDK на Python с удобным API
- Обработка ошибок и разрывов соединения
